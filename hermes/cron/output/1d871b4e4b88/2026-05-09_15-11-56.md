## Root Cause of Previous Failure

The script timed out because **SSH authentication to GitHub wasn't configured for cron/scheduled jobs:**

- An SSH key (`~/.ssh/github_auto`) exists and is valid — I verified it authenticates successfully to GitHub
- But the **ssh-agent wasn't running**, so `git push` hung indefinitely waiting for authentication
- The script's `set -e` + `git push 2>&1 || echo "push failed"` didn't help because SSH blocks forever rather than returning a failure code

## Fix Applied

I started `ssh-agent`, loaded the key, and ran the sync with `SSH_AUTH_SOCK` exported. For future runs, you should either:

1. **Create `~/.ssh/config`** to use the key directly (bypassing ssh-agent):
   ```
   Host github.com
     HostName github.com
     IdentityFile ~/.ssh/github_auto
     IdentitiesOnly yes
   ```

2. **Or persist the ssh-agent** in the cron job by adding `eval "$(ssh-agent -s)" && ssh-add ~/.ssh/github_auto` before the sync command in the cron entry.

All three directories (`~/.hermes/`, `~/.learnings/`, `~/agentic-stack/`, `~/data/`) are now synced to `~/Hermes/` and pushed to GitHub.
