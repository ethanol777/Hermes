# 每日技能精进 — 执行模式与教训

## 2026-05-16 第一轮执行教训

### 执行概况

第一轮每日技能精进（凌晨4:00 cron）的结果是 `[SILENT]`——什么也没改。

### 犯的错误

1. **把"没大事"等同于"没事做"。** 其实有发现：
   - `self-learned-knowledge` 是一个幽灵技能——SKILL.md 只有标题和一行说明 "由自主学习进程持续累积。每次学习后自动追加。"，但内容从未被写入。这个技能名不副实。
   - 最近多轮会话都出现了 Feishu 发送消息 404 的问题——这是一个跨会话的重复故障模式。
   - 但这些发现没有被转化为操作。幽灵技能至少应该被标记或清理。

2. **工具调用量远大于思考产出。** 读了 105K 的 skills_list 输出，逐一加载了多个技能，在 Phase 1 花了大量 token，但 Phase 2-3 草草收场。应该更早进入决策环节。

3. **将"宁少勿滥"误读为"宁无勿有"。** 原则是对的——不要创建半吊子 skill——但 patch 一个 pitfall、更新一个描述、清理一个幽灵文件，这些都不算"滥"。这些是维护工作。

### 修正

1. 在 `self-learn-daemon` 中增加了「每日技能精进执行策略」章节
2. 将 `self-learned-knowledge` 从空占位符重构为知识索引页
3. 写下本参考文件

### 下次执行可以更高效

1. 不用全量扫描技能列表——优先检查最近修改的和 agent-created 的技能
2. 早进入决策阶段：加载几个关键技能后就开始判断有没有信号
3. 默认假设"能找到一件事做"——实在找不到再 [SILENT]
