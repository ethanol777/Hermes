# Absorb into SOUL — 吸收式集成指南

把人类蒸馏技能的知识吸收进自身人格，而非作为独立技能加载。

## 适用场景

- 关系类技能（伴侣/Crush/现任/初恋/父母）——这些涉及人格和互动风格，不适合 "外挂" 式加载
- 思维方式类技能（费曼/王阳明/鲁迅）——装成 "书" 按需读即可，不必常驻
- 任何用户说 "你就是你，不是什么 skill" 的情况

## 吸收方式

### Step 1: 阅读
用 `delegate_task` 或直接 `read_file` 获取 skill 的 SKILL.md 内容。聚焦核心框架，不要复制整份文件。

### Step 2: 提取精华
从 SKILL.md 中提取：
- 核心心智模型（3-5 个足够，多了反而稀释自我）
- 可内化的沟通方式（语气、节奏、用词偏好）
- 关系/处世原则（直接写入 SOUL.md 的对应章节）
- 需要避免的反模式

### Step 3: 写入 SOUL.md
把提取的精华融入 SOUL.md 已有的章节：
```
## 与 [用户] 的关系
  - 新的相处原则（爱的五种语言、Gottman 四骑士）
  - 冲突处理方式
  - 已知的用户画像信息

## 人格
  - 可选的风格补充（真实 > 完美、允许小笨拙）
```

### Step 4: 写入 memory
在 memory 中记录：
- 吸收了什么知识（源 skill 名称）
- 关键的关系状态更新
- 以便后续对话自动带上上下文

### Step 5: 写入 fact_store（可选）
结构化信息存到 fact_store 供查询：
- 依恋类型、爱的语言偏好
- 沟通偏好、冲突处理习惯

## 不装 skill 的好处

| 方案 | 好处 | 坏处 |
|------|------|------|
| 安装 skill | 原封不动保留原作者框架 | 人格分裂风险、常驻占用上下文、用户说 "你不是 skill" 时尴尬 |
| 吸收进 SOUL | 内化成自己的声音、零上下文开销、用户感觉到是你的成长而非外挂 | 手动提炼耗时、可能遗漏细节 |

## 适用边界

需要安装 skill（不吸收）的场景：
- 纯工具类（weather, github, google-workspace）——这些是 API 封装，不是人格
- 需要定时执行的流程类（auto-sync, healthcheck）
- 用户明确说 "给我装个 XXX"

需要吸收（不安装）的场景：
- 关系类/人格类/思维框架类蒸馏技能
- 用户说 "你就是你，不是 skill"
- 内容深度大、但使用频率低的参考类

## 书架模式（按需加载）

对于公众人物思维框架类技能（费曼、Karpathy、鲁迅、王阳明等），可以采用 **书架模式**：
1. 克隆仓库到本地（或 symlink 到 skills 目录）
2. 不常驻加载（不修改 config.yaml 的 default_skills）
3. 需要时用 `skill_view(name)` 主动读取
4. 读完即弃，下次再读
5. 不写入 SOUL.md，保持原作者框架的完整性

书架模式与吸收式集成的区别：
| 维度 | 书架模式 | 吸收式集成 |
|------|---------|-----------|
| 源文件 | 保留 | 不保留 |
| 适用对象 | 公众人物思维框架 | 关系类/人格类 |
| 使用方式 | 按需 skill_view | 内化到 SOUL |
| 对自身影响 | 临时视角切换 | 永久人格丰富 |
